## Требования

### Для Linux
- **Операционная система:** Ubuntu 20.04/22.04/24.04 или другой дистрибутив
- **Компилятор:** GCC с поддержкой C++17
- **CMake:** версия 3.10 или выше
- **Библиотеки:** SFML 2.6.x, nlohmann/json

### Для Windows
- **Операционная система:** Windows 10/11
- **Компилятор:** MinGW-w64 или Visual Studio 2022
- **CMake:** версия 3.10 или выше
- **Библиотеки:** SFML 2.6.x (статическая сборка), nlohmann/json

## Установка и сборка

### Сборка под Linux

#### 1. Установка зависимостей

```bash
sudo apt update
sudo apt install build-essential cmake libsfml-dev nlohmann-json3-dev
```

#### 2. Клонирование репозитория
```bash
git clone https://github.com/your-username/solar-system-sim.git
cd solar-system-sim
```

#### 3. Сборка проекта
```bash
cmake -S . -B build
cmake --build build
```

Исполняемый файл появится в `build/`.

### Сборка под Windows

#### Способ 1: MinGW-w64

##### 1. Установка MinGW-w64
Скачайте и установите MSYS2.

Откройте терминал MSYS2 и выполните:
```bash
pacman -S mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-nlohmann-json
```

##### 2. Установка SFML
Скачайте SFML для MinGW (выберите GCC 13.1.0 MinGW (SEH) - 64-bit)

Распакуйте архив, например, в C:/lib/SFML-2.6.1

##### 3. Настройка переменных окружения
```bash
export SFML_DIR="/c/lib/SFML-2.6.1/lib/cmake/SFML"
```

##### 4. Сборка проекта
```bash
cmake -S . -B build -G "MinGW Makefiles"
cmake --build build
```

Исполняемый файл появится в `build/` (статически слинкованный — не требует DLL).

#### Способ 2: Visual Studio 2022

##### 1. Установка зависимостей через vcpkg
```bash
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
./bootstrap-vcpkg.bat
./vcpkg install sfml nlohmann-json
```

##### 2. Сборка проекта
```bash
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=./vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release
```

Исполняемый файл появится в `build/Release/.`

## Запуск
### Linux
```bash
./build/solar_sym
```

### Windows (MinGW)
```bash
./build/solar_sym.exe
```

### Windows (Visual Studio)
```bash
./build/Release/solar_sym.exe
```

Важно: Программа ожидает наличия директории `configs/` с конфигурационными файлами:
- `configs/obj_config.json` — параметры планет и спутников
- `configs/cam_config.json` — начальные настройки камеры
- `configs/ui_config.json` — настройки шрифтов интерфейса

Примеры конфигурационных файлов находятся в репозитории.

## Управление

### Управление камерой

| Действие | Клавиша/Мышь |
|----------|--------------|
| Вращение камеры (влево/вправо) | ← / → |
| Вращение камеры (вверх/вниз) | ↑ / ↓ |
| Приближение / отдаление | Колёсико мыши |
| Точное движение (замедленное) | Shift + ... |
| Выбор объекта (камера наведётся на него) | Левый клик мыши по объекту |

### Управление временем

| Действие | Кнопка интерфейса |
|----------|-------------------|
| Замедление времени (×0.5) | `<` |
| Ускорение времени (×2) | `>` |
| Нормальная скорость (×1) | — |
| Пауза / Возобновление | `II` |

### Прочие действия

| Действие | Кнопка интерфейса |
|----------|-------------------|
| Показать / скрыть названия объектов | `Hide names` / `Show names` |
| Перезагрузка конфигураций | `Reload` |
| Выход из программы | `x` |

## Интерфейс

### Верхняя панель

Расположена в верхней части экрана. Содержит кнопки управления временем и отображением:

| Кнопка | Функция |
|--------|---------|
| `<` | Замедлить время |
| `II` | Пауза |
| `>` | Ускорить время |
| `x1` / `x2` / `x0.5` | Индикатор текущей скорости симуляции |
| `Hide names` | Скрыть названия объектов |
| `Reload` | Перезагрузить конфигурации |
| `x` | Закрыть программу |

### Панель справки (правый нижний угол)

Сворачиваемая панель с кнопкой `?`. Содержит напоминание об основных клавишах управления.

### Панель редактирования (левый нижний угол)

Сворачиваемая панель с кнопкой `Exp`. Доступна **только на паузе**.

## Редактирование объектов

### Как редактировать существующий объект

1. Нажмите `II` или `Space`, чтобы поставить симуляцию на паузу
2. Нажмите на объект левой кнопкой мыши — он станет выбранным (камера наведётся на него)
3. Разверните панель `Exp` в левом нижнем углу
4. Измените нужные параметры (имя, позиция, скорость, масса, размер, цвет)
5. Нажмите кнопку `Save` — изменения применятся к объекту

### Как создать новый объект

1. Поставьте симуляцию на паузу
2. Заполните поля в панели `Exp` параметрами нового объекта
3. Нажмите `New` — объект появится в симуляции и станет выбранным

### Как удалить объект

1. Поставьте симуляцию на паузу
2. Выберите объект кликом мыши (или он уже выбран)
3. Нажмите `Remove` — объект будет удалён

### Параметры объектов

| Поле | Описание | Единица измерения |
|------|----------|-------------------|
| Name | Название объекта | текст |
| Position X, Y, Z | Координаты в пространстве | а.е. (астрономические единицы) |
| Velocity X, Y, Z | Компоненты скорости | а.е./день |
| Mass | Масса объекта | масса Земли |
| Size | Визуальный размер (для отображения) | а.е. |
| Color R, G, B | Цвет объекта (красный, зелёный, синий) | 0–255 |

## Конфигурационные файлы

### obj_config.json

Содержит массив объектов со следующими полями:

```json
{
    "name": "Earth",
    "position": [1.0, 0.0, 0.0],
    "velocity": [0.0, 0.0172, 0.0],
    "mass": 1.0,
    "size": 4.26e-5,
    "color": [65, 105, 225]
}
```

### cam_config.json

Начальные параметры камеры:

```json
{
    "yaw": 0.0,
    "pitch": 90.0,
    "distance": 15.0,
    "fov": 60.0,
    "rotationSpeed": 3,
    "zoomSpeed": 5,
    "smoothness": 0.85,
    "showNames": true,
    "fontPath": "/usr/share/fonts/truetype/dejavu/DejaVuSans.ttf",
    "characterSize": 17,
    "minShow": 3.5,
    "iconSize": 6
}
```

### ui_config.json

Настройки шрифтов для интерфейса:

```json
{
    "fontPath": "/usr/share/fonts/truetype/dejavu/DejaVuSans.ttf"
}
```
